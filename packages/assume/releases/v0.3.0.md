# v0.3.0

Released: 2026-04-07

## Changes

### New
- Add GCP provider: OAuth 2.0 device flow login, project listing via Cloud Resource Manager API, access token vending via metadata server emulation (port 9912)
- GCP auto-refresh: daemon refreshes GCP access tokens on the same 60s interval as AWS
- GCP console URL: `gsa console <project>` opens GCP Cloud Console for the project
- GCP auto-tagging: projects with "prod", "production", "prd", or "live" in the name are tagged as dangerous
- Interactive GCP setup: `gsa login gcp` prompts for OAuth client ID/secret if not configured
- Provider prompt on login: `gsa login` sets the shell prompt to `[provider:no-profile]` or auto-selects if single context

### Changed
- GCP provider is now enabled by default (was opt-in) — `gsa login gcp` triggers interactive setup on first use
- Help text now mentions both AWS and GCP in the about line for discoverability
- Auto-update now re-execs the current command after upgrading so changes apply immediately (no restart needed)
- `gsa use` now requires provider as first arg: `gsa use gcp <profile>` or `gsa use aws <profile>`
- Shell wrapper evals both `gsa login` and `gsa use` output
- Context exports extracted into shared `print_context_exports()` function

### Fixed
- Make gs-assume resilient to credential endpoint failures
- Require valid session for `gsa use` to prevent stale credential usage
