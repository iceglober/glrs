# v0.1.0

Released: 2026-04-04

Initial release of `gs-assume` (`gsa`) — a Rust-based multi-cloud SSO credential manager.

## New

- **SSO login** via AWS Identity Center device flow (`gsa login aws`)
- **Per-shell context switching** with `gsa use` — each terminal targets its own account/role
- **Credential daemon** — background HTTP server vends auto-refreshing STS credentials via `AWS_CONTAINER_CREDENTIALS_FULL_URI`
- **TUI picker** — interactive fuzzy-search context picker with provider grouping and dangerous-context warnings
- **Shell integration** — `eval "$(gsa shell-init zsh)"` for bash, zsh, fish with prompt and wrapper
- **Auto-upgrade** — minor/patch versions auto-download from GitHub releases; major versions warn
- **Agent access** — permission-gated credential access for AI agents:
  - `gsa agent allow` — TUI multi-select to toggle which contexts agents can access (default deny)
  - `gsa agent exec -- <cmd>` — run commands with daemon endpoint credentials (auto-refreshing)
  - `gsa agent mcp` — JSON-RPC 2.0 MCP server with `run_with_credentials` and `list_contexts` tools
- **Config management** — `gsa config` for interactive setup, team config merging
- **Audit logging** — all credential operations logged to `~/.config/gs-assume/audit.log`
- **AES-256-GCM encrypted vault** for token storage at rest
- **Cross-platform binaries** — macOS (amd64/arm64) + Linux (amd64/arm64)
