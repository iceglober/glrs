# v0.2.0

Released: 2026-04-07

## Changes

### New
- Add GCP provider: OAuth 2.0 device flow login, project listing via Cloud Resource Manager API, access token vending via metadata server emulation (port 9912)
- GCP auto-refresh: daemon refreshes GCP access tokens on the same 60s interval as AWS
- GCP console URL: `gsa console <project>` opens GCP Cloud Console for the project
- GCP auto-tagging: projects with "prod", "production", "prd", or "live" in the name are tagged as dangerous
- Interactive GCP setup: `gsa login gcp` prompts for OAuth client ID/secret if not configured
- Provider prompt on login: `gsa login` sets the shell prompt to `[provider:no-profile]` or auto-selects if single context

### Changed
- `gsa use` now requires provider as first arg: `gsa use gcp <profile>` or `gsa use aws <profile>` (was `gsa use <pattern>`)
- Shell wrapper evals both `gsa login` and `gsa use` output (was `use` only)
- GcpProvider uses `from_config()` constructor (matching AwsProvider pattern)
- GCP env vars set on context switch: `GOOGLE_CLOUD_PROJECT`, `CLOUDSDK_CORE_PROJECT`, `GCE_METADATA_HOST`
- Context exports extracted into shared `print_context_exports()` function used by both `use` and `login`
