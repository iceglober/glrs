# v2.7.0

Released: 2026-04-11

## Changes

### New
- Role-specific preambles (TASK/REVIEW/BUILD) for gs- skills — each skill gets only the state mutations it needs
- Lean compact task output (`listTasks({ lean: true })`) — returns only id, title, phase + non-null fields, skips transitions/children queries
- `--summary` flag on `state review list` for compact review item output
- SQL-based `reviewSummary` using GROUP BY instead of loading full payloads

### Changed
- All `--json` handlers now emit compact JSON (no pretty-printing) to reduce token usage
- gs- skills use role-specific preambles with field projections instead of full task dumps
- `state epic show --json` and `state epic list --json` return only essential fields
- `state task list --json` automatically uses lean mode

### Fixed
- REVIEW_PREAMBLE resolve command: `--id` corrected to `--item`, `--sha` to `--commit-sha`, added missing `--status`
- `state review summary` blocking items now scoped by `--task` filter
- `gs-think.ts` stale reference to removed `gs-agentic start` command
- E2E install-skills tests updated for gs- prefixed skill filenames

### Removed
- `start` command and pipeline orchestrator (`start.ts`, `pipeline.ts`, `session-runner.ts`)
- Classic skills (`/think`, `/work`, `/fix`, `/qa`, `/ship`) — superseded by gs- versions
- `pipelines` database table
- `--parent` backward-compat alias on `state task create` and `state task list`
- `add-workstream` backward-compat alias on `state spec`
