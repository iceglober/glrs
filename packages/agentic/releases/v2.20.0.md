# v2.20.0

Released: 2026-04-13

## Changes

### New
- Auto-sync skills on CLI startup when installed manifest version doesn't match binary version
- Auto-install skills to `~/.claude/` on first CLI run (no manual `gsag skills` needed)
- `skills.auto-update` config setting to disable auto-sync (`gsag config set skills.auto-update false`)
- Version and prefix fields stamped in `.glorious-skills.json` manifest for upgrade detection
- `okErr()` helper in fmt.ts for stderr output with green checkmark

### Changed
- `gsag upgrade` now says "skills will auto-sync on next run" instead of requiring manual `gsag skills`
- Skill sync messages written to stderr instead of stdout to avoid corrupting `--json` output
- `readManifest` validates JSON shape — gracefully handles malformed manifest files

### Fixed
- Prefix roundtrip asymmetry: `""` no longer collapses to `undefined` via `||` coercion (now uses `??`)
- `syncScope` returns `true` when version mismatch is resolved, even if file content was already up-to-date
- Test suites now clean up temp directories in `afterEach`
