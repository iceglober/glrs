# v2.17.0

Released: 2026-04-13

## Changes

### New
- Complete redesign of `gsag state web` dashboard with dark mode UI
- Dashboard overview with summary stats bar (epics, active tasks, blocked, ready, open reviews)
- Segmented progress bars showing task distribution across phases per epic
- Phase stepper visualization (connected circles) in epic and task detail views
- Repo pill selector with health dots for multi-repo mode
- Slide-out task detail panel with review items, markdown-rendered plans, dependency pills
- Sidebar search/filter, active/completed grouping with collapse
- Keyboard navigation: j/k (epics), Enter (select), Esc (back), / (search), 1-9 (repos), ? (shortcuts)
- "Ready to work" section showing tasks with all dependencies met
- Recent activity timeline showing latest phase transitions
- Auto-detect multi-repo: `--all` defaults on when multiple repos exist in DB
- `--local` flag to force single-repo mode
- `GET /api/state/summary` endpoint for lightweight dashboard stats
- `GET /api/task/:id/reviews` endpoint for review items in task detail
- `recentTransitions` included in `/api/state` response
- `stateSummary()` and `listRecentTransitions()` query functions in state layer
- `state task note --id <id> --body "..."` — attach notes to tasks for persistent context
- `state task notes --id <id> [--json]` — list notes for a task
- `state task transition --ids <comma-list> --phase <phase>` — batch transition multiple tasks
- `state task update --id <id> --depends-on <comma-list>` — set/replace task dependencies
- `state plan sync --stdin` — atomically create epic + tasks from a line-based format
- `epicProgress()` — task count breakdown by status (done/cancelled/inProgress/blocked/ready)
- `progressBar()` — visual progress bar for terminal output
- `status --epic <id>` — filter status view to a single epic with progress bar

### Changed
- Dark mode color palette: #0a0a0a backgrounds, desaturated HSL phase colors
- Phase pills use 15% opacity backgrounds with matching text color
- Markdown rendering via marked.js CDN for plan content
- Sidebar now groups epics by active/completed with progress percentages
- TASK_PREAMBLE and BUILD_PREAMBLE now document batch transitions, task notes, plan sync, and progress
- `/deep-plan` Step 7c offers atomic `plan sync` as preferred option over individual `add-task` calls
- `ship` phase now classified as in-progress in epic progress tracking
- Progress bar counts cancelled tasks as resolved (not as incomplete)
