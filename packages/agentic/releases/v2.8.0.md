# v2.8.0

Released: 2026-04-11

## Changes

### New
- **Steps as DB entities** — `steps` table with `s1`/`s2` IDs, phase tracking, sort_order; full CRUD (`createStep`, `loadStep`, `listSteps`, `saveStep`)
- **Versioned plans** — plans stored at `~/.glorious/plans/<repo>/<id>/v1.md`, immutable versions auto-incremented on each save
- **`gsag state plan` command group** — replaces `gsag state spec`; adds `history` subcommand to list plan versions
- **`plan set` works for epics** — no longer restricted to task IDs only
- **`plan_version` columns** on epics and tasks tables, tracking which version is current

### Changed
- Plan storage moved from per-repo `.glorious/specs/` to global `~/.glorious/plans/<repo-slug>/`
- `state spec show/set/add-task` renamed to `state plan show/set/add-task`
- All gs-* skill templates reference `plan` instead of `spec` (plan show, plan set, plan add-task, planContent)
- TASK_PREAMBLE field list updated: `spec` → `plan`
- Entity hierarchy in CLAUDE.md: Epic > Task > Step (steps now tracked in DB)

### Removed
- `state spec` command group (replaced by `state plan`)
- `.glorious/specs/` per-repo storage convention
