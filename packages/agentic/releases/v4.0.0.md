# v4.0.0

Released: 2026-04-19

Worktree management overhaul. All worktrees now live in a global store and the `wt` command surface is simpler.

## Breaking

- **Worktree storage moved** to `~/.glorious/worktrees/<repo>/<name>/` (overridable via `GLORIOUS_DIR` → `$GLORIOUS_DIR/<repo>/<name>`). Previously worktrees were created as siblings of the current repo (`../<name>`). Existing registered worktrees keep working; new ones land in the global store.
- **`wt create` renamed to `wt new`** (`create` kept as an alias). The positional argument is now an optional **repo name** (for creating from outside a git repo) instead of a branch/worktree name — names are auto-generated.

## New

- **`wt path [<name>|<repo>/<name>]`** — print a worktree's absolute path on stdout. With no arg, prints a tab-separated `<repo>/<name>\t<path>` listing for fzf piping. Shell shortcut: `gswt() { cd "$(gs-agentic wt path "$1")"; }`.
- **Auto-generated worktree names** — `gs-agentic wt new` with no args produces `wt-YYMMDD-HHMMSS-xxx` (random 3-char suffix prevents same-second collisions).
- **Create from outside a git repo** — `gs-agentic wt new <repo>` resolves the source repo path from the global registry.

## Changed

- `wt new` always fetches `origin/<default>` before creating and bases the new branch off the freshly-fetched remote ref, never the local one. A stale local `main` can't leak in.
- `defaultBranch()` refactored to delegate to a new `defaultBranchIn(repoPath)`, letting outside-the-repo operations detect the default branch of an arbitrary repo.
