# v4.1.0

Released: 2026-04-19

Worktree UX polish — interactive switching, cross-repo resolution, and nested-worktree guards.

## New

- **`wt switch` (alias `sw`)** — explicit interactive worktree picker. Same as bare `gs-agentic wt`, but works where the bare form is ambiguous (scripts, chained commands).
- **`wt list -i` / `--interactive`** — flip `ls` into the picker without a separate subcommand.
- **Cross-repo resolution for `wt new <name>`** — resolves a repo name from outside a git repo via three tiers: (1) worktree registry, (2) new repo index at `~/.glorious/repos.json` (auto-populated on every gsag invocation inside a git repo), (3) filesystem scan of the configured `repo.scan-roots` (default `~/repos:~/code:~/src`, depth-limited, skips `node_modules`/`target`/`dist`/`vendor`/hidden dirs).
- **`repo.scan-roots` setting** — colon-separated paths to scan when resolving `wt new <repo>` from outside a git repo.
- **`wt protect`** — installs a global git `post-checkout` hook at `~/.glorious/git-hooks/` that warns on any nested worktree creation (even from raw `git worktree add`). Sets `core.hooksPath` globally, with `--force` opt-in if already set elsewhere.
- **`assertPrimaryClone`** — gsag itself now refuses to create a worktree from inside a linked worktree, with a clear error.

## Changed

- **`wt list` shows the checked-out branch** — each row displays the live `HEAD` branch; when it differs from the registered branch, both appear as `registered → current`.
- **Interactive pickers** (bare `wt`, `wt switch`, `wt list -i`) show the same `→ current-branch` suffix when the worktree has switched branches since creation.
- **Failure message for unknown repo names** now names every lookup location checked and prints the exact `config set` command to fix.

## Fixed

- **`runHook` crash outside a git repo** — `gs-agentic wt new <repo>` from a non-repo cwd (e.g. `~/.glorious/worktrees/kn-eng`) no longer throws `fatal: not a git repository` from a hidden `gitRoot()` call. Hooks now resolve their file path from the passed `REPO_ROOT`, never from cwd.
- **Duplicated `currentBranch` helpers** collapsed into `currentBranchIn` in `lib/git.ts`.
- **Startup `gitRoot()` probe** gated behind a cheap `hasGitAncestor(cwd)` filesystem check — no stray git spawns from non-repo directories.
