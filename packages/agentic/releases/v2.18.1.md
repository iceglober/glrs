# v2.18.1

Released: 2026-04-13

## Changes

### Fixed
- `closeAndClaimNext` now accepts a target phase parameter — cancelled tasks no longer silently become done
- Claim enforcement tightened: different actors can no longer re-implement a claimed task without `--force`
- Preamble docs corrected to note terminal transitions bypass claim checks
- `exec()` replaced with `execFile()` in plan-review to prevent command injection
- Build-loop uses atomic `--close-and-claim-next` instead of separate transition + claim
- `qaResult` non-null assertions replaced with safe `?? ""` fallback
- Lean `qaResult` output now includes `timestamp` field
- `||` replaced with `??` in `syncCreateEpicWithTasks` actor resolution
- `ORDERED_PHASES` derived from `PHASES` constant to prevent drift
- State server returns 404 for missing plans instead of 200 with null
- State server error responses no longer expose filesystem paths
- Removed unused `ReviewSummaryResult` import from state-server
- State-html API URL uses `127.0.0.1` to match server bind address
- Plan-server POST /api/feedback enforces 1MB body size limit
- `marked` output sanitized to strip `<script>` tags and `on*` event handlers
- `progressBar` clamped to prevent negative `.repeat()` when done > total
- Last-touched tests isolated from production `~/.glorious/.last-task` file

### Added
- Migration tests for `claimed_by`, `claimed_at`, and `ephemeral` columns
- CLI command integration tests using `execaSync`
- `setLastTouchedPath()` test override following `setPlansDir` pattern
