# v2.6.0

Released: 2026-04-11

## Changes

### New
- Global SQLite state database (`~/.glorious/state.db`) replaces per-repo JSON files
- Epic > Task > Step hierarchy with `state epic create/show/list` commands
- `state task current` — auto-resolve task by worktree/branch
- `state task next --epic <id>` — find next ready task in an epic
- `ready --all` — show ready tasks across all repos
- Review system: `state review create/add-item/resolve/list/summary`
- 11 embedded gs- skill commands distributed via `gsag skills`
- JSON-to-SQLite one-time migration from `.glorious/state/`
- `wt root` command documented in help text

### Changed
- `state task update` accepts specific flags (`--title`, `--description`, `--branch`, `--worktree`, `--pr`)
- `state spec add-workstream` renamed to `add-task` (old name kept as alias)
- Compact task preamble for skills (`gsag state task current --json --with-spec`)
- Help text updated with complete flag documentation for all commands
- `status` command displays epic > task hierarchy with derived phase

### Fixed
- CRITICAL: Infinite recursion in pipeline when task belongs to an epic
- Foreign key enforcement enabled (`PRAGMA foreign_keys = ON`)
- `resolveReviewItem` null guard for nonexistent item IDs
- `persistDb` writes to the init path, not the default path (test isolation)
- Skill files (gs-ship, gs-think, gs-fix) reference valid CLI flags
