# v2.12.0

Released: 2026-04-11

## Changes

### New
- Add `--claim <actor>` flag to `gsag state task next` for atomic task claiming
- Add `reloadDb()` to re-read SQLite DB from disk into the in-memory singleton
- Add `withDbLock()` for advisory file locking around concurrent DB operations
- Stale lock detection: auto-cleanup when lock holder PID is dead or lock is >10s old

### Fixed
- Fix race condition where parallel agents both claim the same task via `task next`
- Fix `plan add-task` creating tasks in `implement` phase instead of `design`
- Fix gs-build Step 2 instructing "Transition to implement" after `--claim` already did it
- Update gs-build-loop intro text and rules to reference `--claim` flag
- Update TASK_PREAMBLE and BUILD_PREAMBLE to include `--claim` in `task next` command

### Changed
- `findNextTask` with `--claim` now acquires a file lock, reloads DB from disk, and uses try/catch instead of redundant `validateTransition` pre-check
- Cache sql.js WASM module for synchronous reuse in `reloadDb`
