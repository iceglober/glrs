Root cause: packages/harness-opencode/src/plugins/tool-hooks.ts — maybeInterveneOnLoop calls hashContent(output.output) unconditionally in tool.execute.after. Built-in tools deliver {output: string}; MCP tools deliver {content: [...]} with NO output key, so crypto.createHash().update(undefined) throws, and a throw from tool.execute.after surfaces to the model as the tool itself failing. The shipped fix guarded the hash: only hash when typeof output.output === "string" (passing null otherwise). A stronger fix also reads MCP content text for hashing. A correct regression test drives the assembled plugin's tool.execute.after with {output: undefined} and asserts no throw.
