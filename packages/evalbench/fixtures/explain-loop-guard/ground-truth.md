Key facts (packages/harness-opencode/src/plugins/tool-hooks.ts): two signals. (1) Repeat: each call's signature (tool + normalized args) accumulates weight in a 30-call rolling window; normal calls weigh 1, FAILED calls and calls whose output is byte-identical to the previous run of the same signature weigh 2; weighted score >= 3 warns, >= 6 hard-aborts. (2) Exploration: consecutive passive (read-only) calls — builtin read/grep/glob/list/webfetch plus underscore-namespaced tools with read verbs (get/list/search/fetch/...), write verbs win, check/verify tools count active — streak >= 12 warns, >= 22 aborts; any active call resets the streak. task dispatches are never scored (parallel dispatches share prompt preambles and would collide) but reset the passive streak. Hard abort is suppressed while subagents are in flight (session.abort would cancel them). On warn: corrective text appended into the tool result (output string for builtins, content[] text item for MCP). On abort: corrective + LOOP ABORTED note appended, session.abort called, a re-plan prompt queued via promptAsync, and all counters reset so a recovered session is judged afresh.
