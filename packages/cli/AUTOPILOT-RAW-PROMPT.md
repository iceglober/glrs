# Task: Write `autopilot-raw.ts` — a standalone OpenCode autopilot runner

Single TypeScript file. Run with `bun autopilot-raw.ts <plan-dir>`. No workspace deps — only `@opencode-ai/sdk`, `yaml`, and `node:*`.

## What it does

1. Reads markdown plan files from `<plan-dir>/` (main.md, wave_0.md, wave_1.md, etc.)
2. Enriches each markdown file into structured YAML specs at `<plan-dir>/spec/` using Opus
3. Executes each unchecked phase using GLM-5, iterating until items are checked
4. Logs every SSE event, tool call, cost update, and error to stderr

## Plan structure

Input (authored by human):
```
<plan-dir>/
├── main.md       # goal, constraints, phase list
├── wave_0.md     # phase 0 acceptance criteria
├── wave_1.md     # phase 1 acceptance criteria
└── wave_2.md     # etc.
```

After enrichment (generated by this script):
```
<plan-dir>/
├── spec/
│   ├── main.yaml   # title, goal, phases with completed flags
│   ├── wave_0.yaml # structured items with checked:true/false, files, verify commands
│   ├── wave_1.yaml
│   └── wave_2.yaml
├── main.md
├── wave_0.md
└── ...
```

`spec/main.yaml` format:
```yaml
title: "Plan title"
goal: "Goal text"
phases:
  - file: wave_0.yaml
    completed: true
  - file: wave_1.yaml
    completed: false
```

Phase YAML format (`spec/wave_N.yaml`):
```yaml
items:
  - id: "1.1"
    intent: "What this item does"
    checked: false
    files:
      - path: src/foo.ts
        isNew: false
    verify: "bun test test/foo.test.ts"
```

## Two-phase execution

### Phase A: Enrichment

For each `.md` file in the plan directory (except scope.md), generate a corresponding YAML spec file. Use agent `"prime"` (routes to Opus). Skip files that already have a spec YAML with all items enriched. Skip `main.md` if `spec/main.yaml` already exists.

The enrichment prompt tells the agent to read the markdown, read relevant codebase files for context, and write the structured YAML spec using the file_edit tool.

### Phase B: Execution

Read `spec/main.yaml` to get the phase list. Skip phases with `completed: true`. For each unchecked phase:
- Create a fresh session
- Send the phase YAML content as a prompt with instructions to complete all unchecked items
- Use agent `"autopilot-fast"` (routes to GLM-5)
- Wait for session.idle, then check if all items are `checked: true`
- Iterate up to 10 times per phase
- Mark phase `completed: true` in spec/main.yaml when all items are checked

## OpenCode SDK critical details

Package: `@opencode-ai/sdk`. Three things matter:

1. **Server:** `createOpencodeServer({ port: 0, timeout: 30_000, hostname: "127.0.0.1" })` — spawns `opencode serve` as a child process.

2. **Client:** `createOpencodeClient({ baseUrl: server.url, directory: process.cwd() })` — the `directory` parameter is MANDATORY. Without it, the SSE event stream only delivers `server.heartbeat` events. No session events will flow. This is the project-scoping mechanism.

3. **SSE event stream:** `client.event.subscribe()` returns `{ stream }` which is an AsyncGenerator. Subscribe BEFORE sending the prompt so you don't miss early events.

Key SDK methods:
- `client.session.create({ query: { directory: cwd }, body: {} })` → returns `{ data: { id: string } }`
- `client.session.prompt({ path: { id: sessionId }, body: { parts: [{ type: "text", text }], agent: "autopilot-fast" } })`
- `client.session.messages({ path: { id: sessionId } })` → returns message history with cost/tokens

SSE event types you'll see:
- `server.connected` — initial connection established
- `server.heartbeat` — keepalive every 10s
- `session.status` — session state changed
- `session.idle` — agent finished (your "done" signal). Filter by `props.sessionID === yourSessionId`
- `session.error` — agent errored. `props.message` has details. Filter by sessionID.
- `message.part.updated` / `message.part.delta` — tool calls and text streaming. `props.part.type === "tool"` means a tool call; `props.part.tool` is the name, `props.part.state.input` has args.
- `message.updated` — message metadata including `props.info.cost` and `props.info.tokens`
- `permission.updated` / `question.asked` — agent asking permission. Auto-reject via `POST ${serverUrl}/question/${props.id}/reject`

## Requirements

- Log EVERYTHING to stderr with timestamps. Every SSE event type, every tool call with its first argument, every error with full props JSON.
- Filter session.idle and session.error by `props.sessionID === yourSessionId` — other sessions' events will also appear on the stream.
- Auto-reject all permission/question events (no human available).
- On session.error: log full props, move to next phase (don't crash).
- Graceful Ctrl+C: close server, exit.
- Idempotent: skip already-enriched files, skip completed phases, resume from where you left off.
